The shell will only look up commands in the directories indicated by the $PATH environment variable. On Unix, the current directory is almost never in the PATH. (I heard somewhere that this is for security reasons.) If you want to run a command in the current directory, use a path to it:

./lab3_name_section < sample_input.txt

See the ./ there? . refers to the current directory.

If you want to be fancy / need to run it over and over again, you can put this in your Makefile:

.PHONY: run
run: lab3_name_section
	./$<

Then just make run, and make will build and run your program all at once.
