Are you sure the file is named lab3_name_section? Unix file systems are case-sensitive, so if it's really Lab3_name_section, that's a different file. A good way to know the command will work is to only type the first part then tab-complete it. Pressing TAB will make the shell look for commands or files that you might want to use and fill in their names for you. Saves a lot of typing.

There's another possibility: the shell will only look up commands in the directories indicated by the $PATH environment variable. On Unix, the current directory is almost never in the PATH. (I heard somewhere that this is for security reasons.) (NOTE: Zeus is one of the "almost never" systems.) If you want to run a command in the current directory, use a path to it:

./lab3_name_section < sample_input.txt

See the ./ there? . refers to the current directory.

If you want to be fancy / need to run it over and over again, you can put this in your Makefile:

.PHONY: run
run: lab3_name_section
	./$<

Then just make run, and make will build and run your program all at once.
